# Contributing

Godex is focused on Go backend engineering guidance for Codex agents.

Good contributions:

- tighten Go backend rules with clear operational value
- improve reference routing from `SKILL.md`
- add concise examples that use public, generic service names
- improve validation or documentation consistency

Avoid:

- company-specific code or private examples
- broad starter-app scaffolding
- unsupported claims about library APIs without current official docs
- abstractions that do not help agents make better Go backend changes

Use Apache-2.0 licensing for contributions.
